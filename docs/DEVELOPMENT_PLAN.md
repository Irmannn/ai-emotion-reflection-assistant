# Codex 开发指令

请根据 PRD 开发项目，但必须按阶段推进，不要一次性实现所有功能。

## 总体要求

技术栈：

- 前端：Next.js + React + TypeScript + Tailwind CSS
- 后端：Python + FastAPI
- 数据库：SQLite
- ORM：SQLModel 或 SQLAlchemy
- AI：通过后端调用大模型 API
- API Key：只能放在后端 `.env` 中

## 开发原则

- 每次只完成一个阶段。
- 每个阶段完成后，先告诉用户改了哪些文件。
- 每个阶段必须给出本地运行和测试方法。
- 不要跳过后端存储。
- 不要把 API Key 写进前端代码。
- 不要实现 PRD 之外的复杂功能。
- 代码要尽量清晰，适合初学全栈的人阅读。
- 对关键代码添加简短注释。
- 每个阶段完成后，等待用户确认再继续下一阶段。

## 阶段 1：项目骨架

### 目标

- 创建前端 Next.js 项目结构
- 创建后端 FastAPI 项目结构
- 配置基础 README
- 前后端可以分别启动
- 后端提供一个健康检查接口

### 验收

- 前端页面能打开
- 后端 `GET /health` 返回正常
- README 说明如何启动前后端

## 阶段 2：SQLite 数据库

### 目标

- 接入 SQLite
- 创建 `reflection_records` 表
- 实现数据库初始化
- 实现历史记录的基础 CRUD 接口

### 接口

- `GET /api/reflections`
- `GET /api/reflections/{id}`
- `DELETE /api/reflections/{id}`
- `POST /api/reflections/{id}/feedback`

### 验收

- 可以创建测试数据
- 可以查询列表
- 可以查询详情
- 可以删除记录
- 可以提交反馈

## 阶段 3：前端表单和历史记录

### 目标

- 实现情绪复盘表单
- 前端生成并保存 `session_id`
- 实现历史记录列表
- 实现历史详情展示
- 实现删除记录

### 验收

- 用户可以填写表单
- 用户可以看到历史记录
- 用户可以删除历史记录
- 前端和后端接口能正常联调

## 阶段 4：接入大模型 API

### 目标

- 后端从 `.env` 读取模型 API Key
- 后端封装模型调用函数
- 根据用户输入生成结构化 Markdown 复盘报告
- 暂时可以先非流式返回

### 验收

- 前端提交表单后，后端能返回 AI 复盘报告
- AI 报告结构符合 PRD
- 生成完成后保存到 SQLite

## 阶段 5：实现流式输出

### 目标

- 后端实现流式响应
- 前端逐步展示 AI 输出内容
- 流式完成后保存完整报告到数据库
- 前端显示生成中、完成、失败状态

### 验收

- 用户点击生成后，可以看到内容逐步出现
- 生成完成后历史记录中能看到完整报告
- 网络或模型异常时有错误提示

## 阶段 6：体验完善

### 目标

- Markdown 渲染
- 表单校验
- Loading / Error / Retry 状态
- 免责声明展示
- 隐私提示
- README 完善

### 验收

- 项目可以作为 MVP 演示
- 页面基本可用
- 代码结构清晰
- README 能指导新用户启动项目
